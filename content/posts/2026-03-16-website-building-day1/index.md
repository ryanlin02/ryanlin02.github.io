## 前言：為什麼要架設這個網站？

這是我從頭開始學習架設個人學習筆記的第一篇紀錄。

身為一個完全沒有程式碼相關經驗、看到英文專有名詞會有點頭痛的新手，我決定挑戰在網路上建立一個完全免費的個人網站。這個名為「慢富」的空間，未來將用來記錄我的理財基礎學習、投資觀念修正，以及近期摸索各種強大 AI 工具（例如輔助生成這個網站的 AI）的心得。

藉由這個網站，希望能把碎片化知識，轉化為有系統的筆記，在未來需要時能隨時複習。

## Day 1 工作日誌：打好地基與裝潢

今天使用我的筆電進行了大量的基礎工程，回顧這幾個小時的進度：

### 1. 開發環境建置
* 成功安裝了 **Homebrew**（雖然一開始遇到路徑報錯，但順利透過終端機指令清除損壞檔案並修復）。
* 透過 Homebrew 順利安裝了網頁框架 **Hugo** 以及版本控制工具 **Git**。
* 完成 **VS Code** 編輯器的設定：安裝中文語系包、綁定 GitHub 帳號（ryanlin02），並確認與 Git 順利連動。

### 2. 網站雛形與雲端部署
* 使用 Hugo 建立了名為 `20260316_blog` 的專案資料夾。
* 成功將本地端的程式碼推送到 **GitHub 遠端倉庫**（ryanlin02.github.io）。
* 經歷了 Jekyll 預設編譯器的報錯後，成功建立 `.github/workflows/hugo.yaml` 自動化腳本，讓 GitHub Actions 機器人順利接手，將網站正式發布到網路上！

### 3. 主題升級與架構優化
* 為了追求「現代簡約、操作流暢」的長期經營目標，果斷卸載了初期的 Ananke 主題，全面改裝為更強大的 **PaperMod** 主題。
* 設定了全新的 `hugo.toml`：開啟日/夜間模式、設定頂部導覽列（分類、標籤、搜尋），並精簡了底部的社群分享按鈕。

### 4. 確立未來的標準寫作流程 (SOP)
* 捨棄舊版的單一檔案模式，全面改用 **Page Bundles（資料夾打包法）**：未來每篇文章都會有專屬的「日期+檔名」資料夾，讓圖文完美綁定，方便未來擴充與管理。
* 重新撰寫了 `archetypes/default.md` 模板，現在每次新增文章，都會自動帶入標準的 Front Matter 格式。

## 結語

雖然看著終端機黑底白字跳出 Error 時難免會緊張，但一步步解題、最終看到自己的網站順利在瀏覽器上亮起綠燈的那一刻，真的是難以言喻的感動。

基礎建設已經完成到一個進度，可以開始來嘗試產出內容了！
